# AI Clicker: 全民皆兵與轉職系統擴充 (Phase 5)

本計畫旨在重構遊戲的人口管理機制，將原本「單純數字的工人」與「獨立招募的英雄」兩個系統整合為一體的 **「動態居民系統」**。同時加入「聖騎士」新職業、隨機居民屬性、以及會自動襲擊城鎮的巨獸機制。

## User Review Required

> [!IMPORTANT]
> 這次更新會**大幅改動人力指派的 UI** 與**底層居民資料結構**。
> 過去您只需點擊 `+` / `-` 就可以分配 50 個伐木工；現在每個居民都是獨立的個體（有名字、屬性、等級），您需要在人力資源面板為他們個別指派工作或送往戰鬥。
> 請確認這樣的「微觀管理」機制符合您對遊戲後期的期望！

## Open Questions

1. **一週的時間定義**：「一週後沒打敗巨獸會破壞建築」，這裡的「一週」是指**現實時間的 7 天**，還是**遊戲內的 7 天**（例如遊戲開著 7 分鐘算一天）？（*本計畫暫定為現實時間 7 天*）
2. **牛馬屬性 (Corporate Slave)**：這個屬性設定為隨機值（例如 0.8x ~ 2.0x）。您希望它只影響「採集資源」的效率，還是也影響「戰鬥時的經驗值獲取/戰鬥力」？（*本計畫暫定為只影響勞動採集效率*）
3. **戰鬥隊伍人數上限**：既然所有居民都可以去打怪，如果我們有 50 個居民，可以「全部 50 人一起上陣打怪」嗎？還是依然維持最多「4人小隊」出擊？（*本計畫暫定為維持 4 人小隊出征，其他人需留在營地工作或待機*）

## Proposed Changes

### 1. 核心資料結構重構 (居民取代數字)
將原本的 `state.workers` (整數) 替換為 `state.population` (陣列)。
每位招募的居民將具備以下屬性：
- `id`: 唯一識別碼
- `name`: 系統隨機生成的姓名
- `level`: 預設 1 級
- `exp`: 經驗值
- `jobClass`: 預設為 "novice" (初心者)。達到 5 級可轉職。
- `faith`: 隨機布林值 (true/false)，決定是否有信仰。
- `slaveStat`: 牛馬屬性 (隨機 0.8 ~ 2.5)，影響勞動產出效率。
- `assignment`: 目前的工作狀態 (idle, woodcutter, miner, combat 等)。

### 2. 轉職系統與新職業「聖騎士 (Paladin)」
- **轉職條件**：居民等級達到 Lv.5。
- **信仰限制**：無信仰者可轉職為：勇者、野蠻人、盾戰、盜賊、弓手、槍手、拳擊手、法師、巫師。有信仰者 (`faith: true`) 才可轉職為：牧師、聖騎士、道士、和尚。
- **聖騎士設計**：
  - 定位：對巨獸特化的高防禦輔助。
  - 特性：擁有嘲諷與治癒能力，且與牧師一樣對巨獸（貪、嗔、癡）具有額外的傷害與減傷加成。

### 3. UI 重構：人力資源頁面 (HR Dashboard)
- 移除原本的 `+` / `-` 指派按鈕。
- 新增一個 **「居民名冊列表 (Roster Table)」**，顯示所有居民的名字、等級、信仰、牛馬屬性。
- 在列表中可以直接透過下拉選單調整每位居民的 `assignment` (分派工作或編入戰鬥小隊)。
- 新增「轉職 (Class Change)」按鈕，當居民滿 5 級時亮起。

### 4. 資源消耗與產出公式調整
- **食量消耗**：被指派工作的居民，每秒會消耗食物。公式為 `基礎消耗 × 等級`。
- **勞動產出**：產出效率公式為 `基礎產出 × 等級 × 牛馬屬性 (slaveStat)`。

### 5. 自動化引發巨獸襲擊 (Boss Invasion)
- 定義「自動化程度」為「擁有的總建築物數量」。
- 當總建築物達到 **50棟** 時，觸發【貪】之巨獸出現；**100棟** 觸發【嗔】；**200棟** 觸發【癡】。
- 巨獸出現時記錄 `timestamp`。如果超過 7 天 (168小時) 尚未在冒險者公會將其擊敗，巨獸將會每小時隨機摧毀一棟建築物。
